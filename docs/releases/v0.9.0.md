# OPSCORE v0.9.0

OPSCORE v0.9.0 is the completed public portfolio MVP for a local-first infrastructure and production-operations incident evidence workbench.

## Highlights

- Structured incident, service and dependency context.
- Sanitized DNS Audit Tool CSV and WATCH v1 imports.
- Bounded single-target DNS, HTTP, TLS and TCP-connectivity evidence collection.
- Deterministic cross-source findings, timeline and missing-evidence guidance.
- Operator-supplied backup-awareness context without backup-platform control.
- Immutable SQLite bundle, analysis and assessment revisions.
- Explicit evidence-backed hypotheses and root-cause assessment.
- Read-only history API and CLI visibility.
- FastAPI service, Typer CLI, Markdown reports and local operator workbench.
- TRACE-aligned professional interface with desktop and narrow-screen workflows.

## Investigation safeguards

OPSCORE does not generate hypotheses, infer root cause or promote assessment status automatically. Confirmed assessments require:

- an explicit conclusion statement;
- supporting evidence;
- no unresolved required-evidence entries.

These controls validate the structure and references of an assessment. They do not independently prove that the conclusion is correct.

## Immutable history

The local history database stores ordered revisions for:

- incident bundles;
- explicit analysis runs;
- accepted operator assessments.

Historical revisions cannot be edited, deleted, restored or rolled back through OPSCORE.

## Safety boundary

OPSCORE remains read-only first. It does not:

- modify production infrastructure;
- control DNS, certificates, services or backup systems;
- crawl sites or scan network ranges;
- retain HTTP response bodies;
- remediate incidents;
- automatically infer root cause;
- rewrite immutable history.

Only synthetic or sanitized evidence belongs in the public repository.

## Upgrade notes

This is the first formal portfolio release. Existing v0.8.0 workspaces are upgraded automatically when the local SQLite history schema is opened. Bundle and analysis history are preserved while assessment revision support is added.

## Quick start

```powershell
.\OPSCORE.ps1 setup
.\OPSCORE.ps1 verify
.\OPSCORE.ps1 demo
.\OPSCORE.ps1 ui
```

The local workbench is available at `http://127.0.0.1:8000`.
